# pipeline/

## Responsibility

- Source ingestion and publishing workspace for Zen Virtual Piano.
- Imports scraped arrangement metadata plus downloaded MIDI assets, then processes queued MIDI sources through conversion, scoring, deduplication, enrichment, and publish workflows.

## Design

- TypeScript ESM workspace with CLI entrypoints in `src/cli.ts` and workers in `src/worker.ts`.
- Import path lives in `src/importers/*` (provider adapters, timestamp matcher, asset upload, catalog write, audit).
- Processing path lives in `src/stages/*` with orchestration in `src/lib/*`.
- DB access is provided through `@zen/db` via `src/lib/runtime-repository.ts`.
- Runtime config and thresholds live in `src/env.ts` and `src/config.ts`.

## Flow

- **Import flow**: `scraper-export.json` + local `.mid` files → normalize provider records → timestamp match → upload/reuse asset → upsert work/arrangement/pipeline job → record import audit
- **Run flow**: `run` / `run --source-items` → inventory queued/running/failed/rejected/published jobs, warn on stranded/stale rows, evaluate stages, then publish, queue review, reject, retry, requeue, or force-generate; lifecycle uses durable `state` plus transient `phase`
- **Async flow**: worker consumes `pipeline.process` jobs and optional AI enrichment follow-up jobs

## Integration

- Depends on `@zen/midi-to-vp`, `@zen/db`, `pg-boss`, `openai`, and storage-backed MIDI assets.
- Revalidates Next.js routes via `SITE_URL` / `REVALIDATION_SECRET` on publish when enabled.
- Consumes scraper exports generated by `midi-scraper-extension` import workflows.
